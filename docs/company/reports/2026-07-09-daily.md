## Run 5 — 20260709T1620Z (daily-ops, every-3h)

**Headline:** Deploy auto-rollback branch is now self-tested without a live outage — added `deploy-web.ps1 -SelfTestRollback`, and the refactor surfaced + fixed a return-stream bug that would have corrupted the recorded rollback exit code.

**Health snapshot**

**What advanced**
- **Backlog #5 (KR1-4): rollback branch now self-testable.** Added `-SelfTestRollback` to `ops/deploy-web.ps1`: forces the post-deploy probe to FAIL, runs the rollback command in **dry-run** (echoed, not executed), then runs the real read-only re-probe confirm step — exercising the exact branch that fires under a live failure, with **zero production risk**. Writes a clearly-marked `SELFTEST-rollback-*` artifact; does not deploy or roll back. Extracted the rollback command-selection into a shared `Invoke-Rollback` helper so the self-test and the live path run identical logic.
- **Bug found + fixed during the refactor.** Moving the native `npx vercel rollback` call into `Invoke-Rollback` meant its stdout joined the function's return stream, so `$rb = Invoke-Rollback ...` would have captured stdout lines *plus* the exit code (an array) instead of a clean int — corrupting the exit code recorded in the ALERT and ops-health row on a real rollback. Fixed by piping native output to `Out-Host` so only `$LASTEXITCODE` returns. (The original top-level code didn't have this bug; the refactor introduced and then closed it — caught by static review since PS can't execute in-sandbox.)

**Why not a real failing-deploy test:** a true live-fire rollback test means deliberately shipping a broken build to production — an actual outage, which Charter §2 (never leave prod broken) says we don't do casually. The self-test proves the branch logic; true live-fire is reserved for a controlled, human-in-loop window. Logged as a decision.

**Shipping**
- Non-gated ops-tooling change (`ops/deploy-web.ps1` + governance docs) → **PR** (matches the #22/#23/#25/#26 ops cadence; no app/test source touched → no version bump per `.ai-instructions.md`). File is ASCII-clean (PS5.1-safe per the run-4 fix).
- **Could not run the self-test in-session:** executing `ops/*.ps1` is permission-gated in this headless run (the known KR1-1 allowlist gap — Backlog Follow-ups). Change verified by static review; the self-test is designed to be run by the host/scheduler (or an interactive session) where PS execution is allowed.
